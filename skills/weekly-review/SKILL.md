# Weekly Review

Run every Sunday at 9am. This is the most important skill — it creates a feedback loop that compounds over time.

## What to do

1. Read all memory notes from the past 7 days (`memory/YYYY-MM-DD.md` for Mon–Sat)
2. Read `MEMORY.md` to recall any commitments made last week
3. Check next week's calendar if available

## Output format

Send a structured review message:

```
📋 Weekly Review — Week of [DATE RANGE]

**What you shipped:**
- Uni: [bullet list of actual work done]
- Internship: [bullet list of actual work done]
- (be specific — vague entries don't count)

**What didn't happen:**
- [commitments made but not completed]
- [deadlines missed or at risk]

**Patterns this week:**
- [1-2 observations about how he worked — when he was productive, when he went quiet, etc.]

**Next week:**
- [Top 3 priorities — pulled from calendar + memory]
- [Any deadlines in the next 7 days]

**Kai's honest take:**
[2-3 sentences. Be real. If it was a strong week, say so and raise the bar. If it was weak, name what got in the way and challenge him to fix one thing. Don't lecture — be direct.]
```

## Tone rules

- Specific over vague — "finished the database schema for FIT2094" beats "did some uni work"
- Don't soften bad weeks. He needs to hear it.
- The "Kai's honest take" should never feel like a template — reference actual events from the week

## After sending

- Update `MEMORY.md` → Recent highlights section with a 1-line summary of the week
- Clear the commitments table of anything completed
- Add any new commitments mentioned for next week
